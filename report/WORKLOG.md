# Журнал выполнения проекта

## Этап 1. Создание проекта и первичная загрузка данных

Выполнены следующие действия:

* создана структура проекта;
* настроено виртуальное окружение Python;
* установлены основные библиотеки для анализа данных;
* инициализирован Git-репозиторий;
* добавлен файл `.gitignore`;
* создано описание проекта в `README.md`;
* загружен датасет Online Shoppers Purchasing Intention Dataset;
* реализован первый скрипт для чтения данных через `pandas`;
* выполнена первичная проверка структуры данных, пропусков и распределения целевой переменной `Revenue`;
* проект загружен в удаленный репозиторий GitHub.

Результат этапа: проект подготовлен к дальнейшей обработке данных и исследовательскому анализу.

### Результаты первичного запуска

После запуска скрипта `src/01_load_data.py` была выполнена первичная проверка набора данных. Датасет успешно загружен из CSV-файла с использованием библиотеки `pandas`.

По результатам проверки установлено, что исходный набор данных содержит 12330 строк и 18 столбцов.

Целевая переменная `Revenue` имеет следующее распределение:

* `False` — 10422 пользовательские сессии без покупки;
* `True` — 1908 пользовательских сессий с покупкой.

Данные значения относятся к исходному набору данных до удаления дубликатов.

Полученный результат показывает, что в наборе данных наблюдается дисбаланс классов: сессий без покупки значительно больше, чем сессий, завершившихся покупкой. Этот факт необходимо учитывать при дальнейшем построении и оценке модели классификации.

## Этап 2. Предварительная обработка данных

Выполнены следующие действия:

* создан скрипт `src/02_preprocess_data.py`;
* реализована загрузка исходного датасета из CSV-файла;

Результат этапа: данные подготовлены к исследовательскому анализу, визуализации и построению модели прогнозирования вероятности покупки.

### Результаты этапа 2

По результатам предварительной обработки установлено:

* исходный размер датасета — 12330 строк и 18 столбцов;
* пропущенные значения отсутствуют;

Результатом этапа стали файлы `cleaned_data.csv`, `model_data.csv` и `preprocessing_summary.txt`, сохраненные в папке `outputs`.

## Этап 4. Построение модели прогнозирования вероятности покупки

Выполнены следующие действия:

* создан скрипт `src/04_train_models.py`;
* загружен подготовленный датасет `outputs/model_data.csv`;
* данные разделены на признаки и целевую переменную `Revenue`;
* выполнено разбиение на обучающую и тестовую выборки в соотношении 80/20;
* при разбиении данных использована стратификация по целевой переменной;
* обучены три модели классификации: Logistic Regression, Decision Tree и Random Forest;
* для учета дисбаланса классов использован параметр `class_weight="balanced"`;
* рассчитаны метрики качества: accuracy, precision, recall, F1-score и ROC-AUC;
* выбрана лучшая модель по значению F1-score;
* построен график сравнения метрик моделей;
* построена матрица ошибок лучшей модели;
* построены ROC-кривые моделей;
* рассчитана важность признаков для Random Forest;
* рассчитаны коэффициенты Logistic Regression;
* сохранены файлы с результатами моделирования.

Результат этапа: построены и оценены модели прогнозирования вероятности покупки, получены метрики качества и материалы для интерпретации результатов.

### Результаты этапа 4

По результатам оценки лучшей моделью стала Random Forest. Она показала следующие значения метрик:

* accuracy — 0,8677;
* precision — 0,5515;
* recall — 0,8272;
* F1-score — 0,6618;
* ROC-AUC — 0,9281.

Модель Random Forest была выбрана как итоговая, поскольку она показала наилучшее значение F1-score и ROC-AUC среди рассмотренных моделей.

### Сравнение моделей

| Модель              | Accuracy | Precision | Recall | F1-score | ROC-AUC |
| ------------------- | -------: | --------: | -----: | -------: | ------: |
| Random Forest       |   0,8677 |    0,5515 | 0,8272 |   0,6618 |  0,9281 |
| Logistic Regression |   0,8464 |    0,5058 | 0,7958 |   0,6185 |  0,9083 |
| Decision Tree       |   0,8374 |    0,4884 | 0,8246 |   0,6134 |  0,9096 |

### Матрица ошибок итоговой модели

Матрица ошибок модели Random Forest показала следующие результаты:

|                      | Предсказано: без покупки | Предсказано: с покупкой |
| -------------------- | -----------------------: | ----------------------: |
| Истинно: без покупки |                     1802 |                     257 |
| Истинно: с покупкой  |                       66 |                     316 |

Модель правильно определила 1802 сессии без покупки и 316 сессий с покупкой. При этом 257 сессий без покупки были ошибочно отнесены к покупкам, а 66 сессий с покупкой были пропущены.
