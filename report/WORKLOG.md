# Журнал выполнения проекта

## Этап 1. Создание проекта и первичная загрузка данных

Выполнены следующие действия:

- создана структура проекта;
- настроено виртуальное окружение Python;
- установлены основные библиотеки для анализа данных;
- инициализирован Git-репозиторий;
- добавлен файл `.gitignore`;
- создано описание проекта в `README.md`;
- загружен датасет Online Shoppers Purchasing Intention Dataset;
- реализован первый скрипт для чтения данных через pandas;
- выполнена первичная проверка структуры данных, пропусков и распределения целевой переменной `Revenue`.

Результат этапа: проект подготовлен к дальнейшей обработке данных и исследовательскому анализу.

### Результаты первичного запуска

После запуска скрипта `src/01_load_data.py` была выполнена первичная проверка набора данных. Датасет успешно загружен из CSV-файла с использованием библиотеки `pandas`.

По результатам проверки установлено, что целевая переменная `Revenue` имеет следующее распределение:

- `False` — 10422 пользовательские сессии без покупки;
- `True` — 1908 пользовательских сессий с покупкой.

Полученный результат показывает, что в наборе данных наблюдается дисбаланс классов: сессий без покупки значительно больше, чем сессий, завершившихся покупкой. Этот факт необходимо учитывать при дальнейшем построении и оценке модели классификации.

## Этап 2. Предварительная обработка данных

Выполнены следующие действия:

* создан скрипт `src/02_preprocess_data.py`;
* реализована загрузка исходного датасета из CSV-файла;
* проведена проверка пропущенных значений;
* выполнена проверка дубликатов;
* логические признаки `Weekend` и `Revenue` преобразованы в числовой формат;
* категориальные признаки подготовлены к машинному обучению с помощью one-hot encoding;
* сохранен очищенный датасет `outputs/cleaned_data.csv`;
* сохранен датасет для машинного обучения `outputs/model_data.csv`;
* сформирован файл с итогами предобработки `outputs/preprocessing_summary.txt`.

Результат этапа: данные подготовлены к исследовательскому анализу, визуализации и построению модели прогнозирования вероятности покупки.


### Результаты этапа 2

По результатам предварительной обработки установлено:

* исходный размер датасета — 12330 строк и 18 столбцов;
* пропущенные значения отсутствуют;
* обнаружено и удалено 125 дубликатов;
* размер очищенного датасета — 12205 строк и 18 столбцов;
* признаки `Weekend` и `Revenue` преобразованы из логического формата в числовой;
* категориальные признаки преобразованы с помощью one-hot encoding;
* размер датасета для модели после кодирования — 12205 строк и 69 столбцов.

Результатом этапа стали файлы `cleaned_data.csv`, `model_data.csv` и `preprocessing_summary.txt`, сохраненные в папке `outputs`.
