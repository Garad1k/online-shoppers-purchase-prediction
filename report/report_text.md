# Краткий отчет по итоговому проекту

## Тема проекта

**Анализ поведения пользователей интернет-магазина и прогнозирование вероятности покупки с использованием Python**

## Цель проекта

Цель проекта — провести анализ поведения пользователей интернет-магазина и построить модель машинного обучения, позволяющую прогнозировать вероятность совершения покупки на основе признаков пользовательской сессии.

В рамках проекта решалась задача бинарной классификации. Целевая переменная `Revenue` показывает, завершилась ли пользовательская сессия покупкой:

* `0` — покупка не совершена;
* `1` — покупка совершена.

## Используемые данные

В качестве исходных данных использован набор **Online Shoppers Purchasing Intention Dataset**, содержащий сведения о пользовательских сессиях интернет-магазина.

Исходный датасет содержал 12330 строк и 18 столбцов. После проверки и удаления 125 дубликатов очищенный набор данных составил 12205 строк и 18 столбцов.

В данных отсутствовали пропущенные значения. Логические признаки `Weekend` и `Revenue` были преобразованы в числовой формат. Категориальные признаки были подготовлены к машинному обучению с помощью one-hot encoding. После кодирования датасет для модели содержал 12205 строк и 69 столбцов.

## Исследовательский анализ

В ходе исследовательского анализа было установлено, что в данных наблюдается дисбаланс классов. Большинство пользовательских сессий не завершается покупкой.

Основные результаты анализа:

| Показатель                                       |    Значение |
| ------------------------------------------------ | ----------: |
| Количество пользовательских сессий после очистки |       12205 |
| Сессии без покупки                               |       10297 |
| Сессии с покупкой                                |        1908 |
| Общая конверсия                                  |      15,63% |
| Месяц с наибольшей конверсией                    |    November |
| Максимальная конверсия по месяцам                |      25,49% |
| Тип посетителя с наибольшей конверсией           | New_Visitor |
| Максимальная конверсия по типу посетителя        |      24,93% |

Корреляционный анализ показал, что наиболее связанным с покупкой числовым признаком является `PageValues`. Коэффициент корреляции данного признака с целевой переменной `Revenue` составил 0,4919.

Также положительная связь с покупкой наблюдается у признаков `ProductRelated`, `ProductRelated_Duration`, `Administrative` и `Informational`. Отрицательная связь выявлена у признаков `ExitRates` и `BounceRates`.

## Построение моделей

Для прогнозирования вероятности покупки были обучены и сравнены три модели классификации:

* Logistic Regression;
* Decision Tree;
* Random Forest.

Данные были разделены на обучающую и тестовую выборки в соотношении 80/20. При разбиении использовалась стратификация по целевой переменной, чтобы сохранить соотношение классов в обучающей и тестовой выборках.

Для учета дисбаланса классов в моделях использовался параметр `class_weight="balanced"`.

## Оценка качества моделей

Качество моделей оценивалось по нескольким метрикам: accuracy, precision, recall, F1-score и ROC-AUC. Использование нескольких метрик необходимо, так как в данных наблюдается дисбаланс классов, и одной метрики accuracy недостаточно для полноценной оценки качества модели.

Результаты сравнения моделей:

| Модель              | Accuracy | Precision | Recall | F1-score | ROC-AUC |
| ------------------- | -------: | --------: | -----: | -------: | ------: |
| Random Forest       |   0,8677 |    0,5515 | 0,8272 |   0,6618 |  0,9281 |
| Logistic Regression |   0,8464 |    0,5058 | 0,7958 |   0,6185 |  0,9083 |
| Decision Tree       |   0,8374 |    0,4884 | 0,8246 |   0,6134 |  0,9096 |

Лучшей моделью по значению F1-score стала **Random Forest**. Она также показала наибольшее значение ROC-AUC.

## Интерпретация итоговой модели

Модель Random Forest показала лучший баланс между точностью и полнотой выявления сессий, завершившихся покупкой.

Матрица ошибок итоговой модели:

|                      | Предсказано: без покупки | Предсказано: с покупкой |
| -------------------- | -----------------------: | ----------------------: |
| Истинно: без покупки |                     1802 |                     257 |
| Истинно: с покупкой  |                       66 |                     316 |

Модель правильно определила 1802 сессии без покупки и 316 сессий с покупкой. При этом 257 сессий без покупки были ошибочно отнесены к покупкам, а 66 реальных покупок были пропущены.

Значение recall модели Random Forest составило 0,8272. Это означает, что модель смогла выявить большую часть реальных сессий, завершившихся покупкой. Для задач интернет-маркетинга это важно, поскольку позволяет не пропускать значительную часть потенциальных покупателей.

Анализ важности признаков показал, что наиболее значимым признаком является `PageValues`. Также важную роль играют `ExitRates`, `ProductRelated_Duration`, `ProductRelated` и `BounceRates`.

## Практическая значимость

Полученные результаты могут использоваться как вспомогательный инструмент интернет-маркетинга. Модель позволяет выделять пользовательские сессии с повышенной вероятностью покупки и использовать эти данные для:

* персонализации предложений;
* настройки ретаргетинга;
* анализа пользовательского пути;
* выявления факторов, влияющих на конверсию;
* поддержки маркетинговых решений.

## Вывод

В рамках проекта был реализован полный цикл анализа данных: загрузка, предварительная обработка, исследовательский анализ, визуализация, построение моделей машинного обучения, оценка качества и интерпретация результатов.

Цель проекта достигнута: построена модель, позволяющая прогнозировать вероятность совершения покупки пользователем интернет-магазина на основе признаков пользовательской сессии.

Итоговой моделью выбрана Random Forest, показавшая F1-score 0,6618 и ROC-AUC 0,9281.
