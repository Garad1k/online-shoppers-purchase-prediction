# Анализ поведения пользователей интернет-магазина и прогнозирование вероятности покупки

## Описание проекта

Проект посвящен анализу поведения пользователей интернет-магазина и построению модели прогнозирования вероятности совершения покупки с использованием Python.

В качестве исходных данных используется набор **Online Shoppers Purchasing Intention Dataset**, содержащий сведения о пользовательских сессиях интернет-магазина. Данные позволяют исследовать поведение пользователей на сайте, выявлять признаки, связанные с покупкой, и строить модель классификации для прогнозирования целевого действия.

## Цель проекта

Провести анализ поведения пользователей интернет-магазина и построить модель классификации, позволяющую прогнозировать вероятность совершения покупки на основе признаков пользовательской сессии.

## Задачи проекта

В рамках проекта были решены следующие задачи:

1. Загрузить исходный набор данных и выполнить первичную проверку его структуры.
2. Провести предварительную обработку данных.
3. Проверить данные на наличие пропусков и дубликатов.
4. Преобразовать логические и категориальные признаки в формат, пригодный для анализа и машинного обучения.
5. Выполнить исследовательский анализ пользовательского поведения.
6. Построить визуализации по ключевым признакам.
7. Обучить и сравнить несколько моделей классификации.
8. Оценить качество моделей по нескольким метрикам.
9. Выбрать итоговую модель и интерпретировать полученные результаты.

## Используемые данные

В проекте используется датасет `online_shoppers_intention.csv`.

Основные характеристики исходного набора данных:

* количество строк: 12330;
* количество признаков: 18;
* целевая переменная: `Revenue`;
* тип задачи: бинарная классификация.

Целевая переменная `Revenue` показывает, завершилась ли пользовательская сессия покупкой:

* `False` / `0` — покупка не совершена;
* `True` / `1` — покупка совершена.

После удаления дубликатов очищенный датасет содержит 12205 строк и 18 столбцов.

## Этапы реализации

### 1. Загрузка и первичная проверка данных

На первом этапе был создан скрипт `src/01_load_data.py`, который выполняет загрузку исходного CSV-файла с помощью библиотеки `pandas`.

На данном этапе были проверены:

* первые строки датасета;
* размер набора данных;
* типы данных;
* наличие пропусков;
* распределение целевой переменной `Revenue`.

Распределение целевой переменной в исходном наборе данных:

| Значение Revenue | Количество сессий |
| ---------------- | ----------------: |
| False            |             10422 |
| True             |              1908 |

Полученный результат показывает наличие дисбаланса классов: сессий без покупки значительно больше, чем сессий, завершившихся покупкой.

### 2. Предварительная обработка данных

На втором этапе был создан скрипт `src/02_preprocess_data.py`.

Выполненные действия:

* проверка пропущенных значений;
* проверка дубликатов;
* удаление 125 дубликатов;
* преобразование признаков `Weekend` и `Revenue` из логического формата в числовой;
* кодирование категориальных признаков с помощью one-hot encoding;
* сохранение очищенного датасета;
* сохранение датасета для машинного обучения.

Результаты этапа:

| Показатель                             |   Значение |
| -------------------------------------- | ---------: |
| Исходный размер датасета               | 12330 × 18 |
| Количество удаленных дубликатов        |        125 |
| Размер очищенного датасета             | 12205 × 18 |
| Размер датасета после one-hot encoding | 12205 × 69 |

После обработки были сформированы файлы:

* `outputs/cleaned_data.csv`;
* `outputs/model_data.csv`;
* `outputs/preprocessing_summary.txt`.

### 3. Исследовательский анализ и визуализация

На третьем этапе был создан скрипт `src/03_exploratory_analysis.py`.

Были рассчитаны:

* количество пользовательских сессий;
* количество сессий с покупкой;
* количество сессий без покупки;
* общая конверсия;
* конверсия по месяцам;
* конверсия по типу посетителя;
* корреляции числовых признаков с целевой переменной `Revenue`.

Основные результаты исследовательского анализа:

| Показатель                                     |    Значение |
| ---------------------------------------------- | ----------: |
| Количество пользовательских сессий             |       12205 |
| Количество сессий без покупки                  |       10297 |
| Количество сессий с покупкой                   |        1908 |
| Общая конверсия                                |      15.63% |
| Месяц с наибольшей конверсией                  |         Nov |
| Максимальная конверсия по месяцам              |      25.49% |
| Тип посетителя с наибольшей конверсией         | New_Visitor |
| Максимальная конверсия по типу посетителя      |      24.93% |
| Наиболее связанный с покупкой числовой признак |  PageValues |
| Корреляция PageValues с Revenue                |      0.4919 |

Построенные графики сохранены в папке `outputs/figures`.

### 4. Построение моделей машинного обучения

На четвертом этапе был создан скрипт `src/04_train_models.py`.

Для прогнозирования вероятности покупки были обучены и сравнены три модели классификации:

* Logistic Regression;
* Decision Tree;
* Random Forest.

Данные были разделены на обучающую и тестовую выборки в соотношении 80/20. Для сохранения соотношения классов использовалась стратификация по целевой переменной `Revenue`.

Для учета дисбаланса классов в моделях применялся параметр `class_weight="balanced"`.

### 5. Оценка качества моделей

Качество моделей оценивалось по следующим метрикам:

* Accuracy;
* Precision;
* Recall;
* F1-score;
* ROC-AUC.

Результаты сравнения моделей:

| Модель              | Accuracy | Precision | Recall | F1-score | ROC-AUC |
| ------------------- | -------: | --------: | -----: | -------: | ------: |
| Random Forest       |   0.8677 |    0.5515 | 0.8272 |   0.6618 |  0.9281 |
| Logistic Regression |   0.8464 |    0.5058 | 0.7958 |   0.6185 |  0.9083 |
| Decision Tree       |   0.8374 |    0.4884 | 0.8246 |   0.6134 |  0.9096 |

Лучшей моделью по значению F1-score стала **Random Forest**.

## Итоговая модель

В качестве итоговой модели выбрана **Random Forest**, так как она показала лучший баланс между точностью и полнотой выявления сессий, завершившихся покупкой.

Основные метрики Random Forest:

| Метрика   | Значение |
| --------- | -------: |
| Accuracy  |   0.8677 |
| Precision |   0.5515 |
| Recall    |   0.8272 |
| F1-score  |   0.6618 |
| ROC-AUC   |   0.9281 |

Матрица ошибок модели Random Forest:

|                      | Предсказано: без покупки | Предсказано: с покупкой |
| -------------------- | -----------------------: | ----------------------: |
| Истинно: без покупки |                     1802 |                     257 |
| Истинно: с покупкой  |                       66 |                     316 |

Модель хорошо выявляет реальные сессии с покупкой: значение recall составляет 0.8272. Это означает, что модель смогла найти большую часть пользователей, которые действительно совершили покупку.

## Важность признаков

По результатам анализа важности признаков модели Random Forest наиболее значимым признаком оказался `PageValues`.

Также важными признаками стали:

* `ExitRates`;
* `ProductRelated_Duration`;
* `ProductRelated`;
* `BounceRates`;
* `Month_Nov`;
* `Administrative`.

Это подтверждает, что вероятность покупки связана с характером взаимодействия пользователя с сайтом: ценностью просмотренных страниц, длительностью просмотра товарных страниц, количеством товарных действий, отказами и выходами со страниц.

```bash
cd online-shoppers-purchase-prediction
```

```bash
python -m venv .venv
```

```bash
.\.venv\Scripts\Activate.ps1
```

